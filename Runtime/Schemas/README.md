### Message Envelope Hierarchy

An ARENA MQTT message is structured as a hierarchy of nested JSON. The `type` field in the message determines which data wrapper to use:

```
ArenaMessageJson                         ← root message (object_id, type, action, ttl, ...)
├── type: "object"
│   └── ArenaDataObjectJson              ← data wrapper (position, rotation, scale, material, ...)
│       ├── object_type geometry         ← Object/ schemas (box depth/height, sphere radius, ...)
│       └── attached attributes          ← Attribute/ schemas (animation, click-listener, ...)
└── type: "scene-options"
    └── ArenaDataSceneOptionsJson        ← data wrapper (env-presets, renderer-settings, ...)
        └── scene-options                ← Attribute/ArenaSceneOptionsJson (NavMesh, Physics, ...)
```

- **`ArenaMessageJson`** (root): The MQTT wire envelope. Contains `object_id`, `type`, `action`, `persist`, `ttl`, and the `data`/`attributes` payload.
- **`ArenaDataObjectJson`** (root): Data wrapper for `type: "object"` messages. Holds `object_type`, transform properties (position, rotation, scale), and references to all attachable attributes.
- **`ArenaDataSceneOptionsJson`** (root): Data wrapper for `type: "scene-options"` messages. Holds references to env-presets, renderer-settings, scene-options (attribute), and post-processing.
- **`Object/`**: Defines the geometry or model for each `object_type` value. Each class has a `public readonly string object_type` field (e.g. `"box"`, `"gltf-model"`, `"light"`).
- **`Attribute/`**: Defines attachable attributes. Each class has a `public readonly string attributeName` field (e.g. `"animation"`, `"material"`, `"click-listener"`). These are composed inside `ArenaDataObjectJson` and can be attached to any object type.
- **`Server/`**: Response schemas for server endpoints (MQTT auth tokens, scene defaults, user state). Not part of the scene object hierarchy.
- **`Converter/`**: Custom `JsonConverter` implementations for handling polymorphic JSON types (string-or-object, boolean-or-object, color formats, vector formats).

All schema classes share the `namespace ArenaUnity.Schemas` regardless of subfolder location. Unity resolves classes by namespace, not file path.

## Why Newtonsoft.Json Instead of Unity's JsonUtility

Unity's built-in `JsonUtility` is the fastest JSON parser available in Unity — it serializes/deserializes C# classes with near-zero allocation overhead. However, `JsonUtility` was designed for round-tripping Unity's own serialized data. ARENA's JSON wire format originates from the A-Frame/three.js ecosystem and uses conventions that are fundamentally incompatible with `JsonUtility`.

### Specific Incompatibilities

1. **Hyphenated property names.** JSON keys like `"click-listener"`, `"video-control"`, and `"goto-url"` are illegal C# identifiers. Newtonsoft's `[JsonProperty(PropertyName = "video-control")]` handles the mapping; `JsonUtility` has no equivalent.

   ```csharp
   [JsonProperty(PropertyName = "video-control")]
   public ArenaVideoControlJson videoControl = null;
   ```

2. **Polymorphic `from`/`to` values.** The `animation` attribute's `from` and `to` fields can be either a string (`"blue"`) or an object (`{"x": 1, "y": 1, "z": 100}`). `JsonUtility` requires a single concrete type per field. Newtonsoft's custom converters (`ArenaStringObjectJsonConverter`) handle this gracefully without throwing exceptions.

   ```json
   "to": "blue"
   ```
   ```json
   "to": {"x": 1, "y": 1, "z": 100}
   ```

3. **Boolean-or-object attributes.** Some attributes like `click-listener` can appear as `true` (shorthand) or as a full object `{"enabled": true}`. The `ArenaBooleanObjectJsonConverter` handles both forms transparently.

4. **Graceful error recovery.** ARENA ingests JSON from many sources (web clients, Python scripts, IoT devices) — "the wilds." Malformed or unexpected fields are common. Newtonsoft's `[OnError]` handler lets us log warnings and continue rather than crash:

   ```csharp
   [OnError]
   internal void OnError(StreamingContext context, ErrorContext errorContext)
   {
       Debug.LogWarning($"{errorContext.Error.Message}: {errorContext.OriginalObject}");
       errorContext.Handled = true;
   }
   ```

5. **Extension data capture.** The `[JsonExtensionData]` attribute preserves unknown JSON fields in a dictionary, so new or custom properties added to the schema don't cause parse failures and can be round-tripped.

6. **Conditional serialization.** The `ShouldSerialize*()` pattern lets us send only changed fields over MQTT, minimizing bandwidth. `JsonUtility` always serializes every field.

### Performance Tradeoffs

| Aspect | JsonUtility | Newtonsoft.Json |
|--------|-------------|-----------------|
| **Parse speed** | ~2–5× faster | Slower due to reflection and converters |
| **Allocations** | Minimal (struct-friendly) | Higher (boxing, dictionary alloc for extension data) |
| **Flexibility** | C#-style JSON only | Arbitrary JSON shapes, polymorphism, custom converters |
| **Error tolerance** | Throws on mismatch | Configurable error handling, can continue on failure |
| **Unknown fields** | Silently dropped | Captured via `[JsonExtensionData]` |
| **Bandwidth** | All fields always | Conditional serialization via `ShouldSerialize*()` |

### Why We Accept the Cost

For ARENA's use case, correctness and resilience outweigh raw parse speed:

- **Ingress from the wilds.** Messages arrive from diverse clients (web, Python, Unity, IoT). The parser must tolerate missing fields, extra fields, type mismatches, and schema evolution without crashing.
- **Bandwidth-sensitive MQTT.** Conditional serialization means we only send what changed — critical for real-time XR over constrained networks.
- **Schema evolution.** New A-Frame components or properties appear regularly. `[JsonExtensionData]` means the Unity client doesn't break when the schema evolves ahead of a client release.

> **Future optimization note:** If large-scale ingress parsing becomes a bottleneck, a hybrid approach could use Newtonsoft for initial parse + validation, then cache converted objects for fast access. The Newtonsoft library shipped with Unity (`com.unity.nuget.newtonsoft-json`) is the officially supported path: https://docs.unity3d.com/Packages/com.unity.nuget.newtonsoft-json@3.2/manual/index.html
